Lab #8 - SameSite Strict bypass via client-side redirect

Vulnerability  - This lab's change email function is vulnerable to CSRF.

Goal - To solve the lab, perform a CSRF attack that changes the victim's email address. You should use the provided exploit server to host your attack.

Credentials - wiener:peter

- Log in with credentials - wiener:peter

<script>redirectOnConfirmation('/post');</script>

redirectOnConfirmation = (blogPath) => {
    setTimeout(() => {
        const url = new URL(window.location);
        const postId = url.searchParams.get("postId");
        window.location = blogPath + '/' + postId;
    }, 3000);
}
