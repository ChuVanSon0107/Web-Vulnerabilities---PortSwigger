Lab: User ID controlled by request parameter with password disclosure

Vulnerability: This lab has user account page that contains the current user's existing password, prefilled in a masked input.

Goal: To solve the lab, retrieve the administrator's password, then use it to delete the user carlos.

Credentials: wiener:peter

Analysis and Solution:
1. Log in with credentials  wiener:peter 
=> In account page, we can see password of this account in HTML response 

2. This websites uses user ID to access user's account: /my-account?id=wiener 
=> Try change to /my-account?id=administrator
=> Now we can access administrator's page 

3. Take the password of administrator and log in with administrator account 
=> Delete carlos account 
=> SUCCESSFULLY
