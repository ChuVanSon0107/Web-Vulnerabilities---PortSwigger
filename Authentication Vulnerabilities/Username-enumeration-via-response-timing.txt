Goal - enumerate a valid username, brute-force this user's password, then access their account page.

Analysis:
- See the response time with a valid username => longer than the response time with an invalid username
- The length of password is much longer => the response time is much longer
- Server will block our IP address if send a lot of incorrect username and password => use X-Forwarded-For in the header of the request.

Solution:
- Use Pithfork attack mode in Intruder of BurpSuite to change X-Forwarded-For and another payloads
- Using Intruder(BurpSuite) to find valid username => archie (with the highest response time)
- Using Intruder(BurpSuite) to find correct password of this username => trustno1
