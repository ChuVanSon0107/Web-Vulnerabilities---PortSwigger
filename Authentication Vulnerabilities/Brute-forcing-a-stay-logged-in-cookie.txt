Analysis:
1. When logging with credentials wiener:peter, see that:
=> if we click "stay logged in", the response contains "stay-logged-in" cookie and session cookie
2. When we send a request to /my-account?id=wiener without session cookie, we successfully access this page
3. When we try to decode this stay-logged-in cookie as base 64, see that the result is wiener:51dc30ddc473d43a6011e9ebba6ca770, 
=> we try to decrypt 51dc30ddc473d43a6011e9ebba6ca770 as MD5, it is peter (password)

Solutions:
1. Create a python file to:
+ encrypt a list of passwords as MD5
+ concatenate string username:{encrypted_password}
+ decode them as base64
2. Using Burp Intruder to brute-force stay-logged-in cookie
=> make a request to /my-account/change-email with brute-force stay-logged-in cookie
==> see that password is 112233
